◆ distributed_client_main()

int distributed_client_main ( int  argc,
char **  argv 
)

Запустить модуль рабочего клиента.

Аргументы
argcЧисло аргументов командной строки.
argvАргументы в том же формате, что и у отдельного бинарника client: argv[0] host port cores worker-id connect-timeout-ms compute-timeout-ms.
Возвращает
EXIT_SUCCESS при успехе, EXIT_FAILURE при некорректном вводе или сетевой ошибке.
Предупреждения
Функция может косвенно вызвать exit() через общие функции разбора аргументов и сетевые helper-ы при обнаружении фатальных ошибок.

◆ distributed_server_main()

int distributed_server_main ( int  argc,
char **  argv 
)

Запустить модуль управляющего сервера.

Аргументы
argcЧисло аргументов командной строки.
argvАргументы в том же формате, что и у отдельного бинарника server: argv[0] points-file num-clients port server-timeout-ms.
Возвращает
EXIT_SUCCESS при успехе, EXIT_FAILURE при некорректном вводе или отказе worker-а.
Предупреждения
Функция блокируется до подключения всех ожидаемых worker-ов, потому что учебный протокол предполагает фиксированный набор worker-ов до раздачи задач.
